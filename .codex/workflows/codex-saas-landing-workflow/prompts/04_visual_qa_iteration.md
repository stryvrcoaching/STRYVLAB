# Prompt 04 — Visual QA iteration

Use the `codex-ui-execution-qa`, `ui-ux-pro-max`, and `landing-visual-ui-motion` skills. If Liquid Glass is active, also use `liquid-glass`.

Open the landing page and perform a strict visual QA pass.

Check:
- desktop;
- tablet;
- mobile;
- hero clarity;
- CTA visibility;
- section hierarchy;
- typography;
- spacing;
- alignment;
- product visual readability;
- proof placement;
- pricing/access clarity;
- FAQ readability;
- motion behavior;
- reduced-motion fallback;
- Framer Motion / GSAP conflicts, transform-only motion, and layout shift;
- Liquid Glass Chromium rendering and Safari/Firefox fallback when active;
- accessibility basics;
- performance risks.

Fix visible issues directly.

Do not redesign the whole page unless necessary.
Do not add new sections unless required by the brief.
Do not invent proof.

Final output:
1. Issues found
2. Fixes applied
3. Screensizes checked
4. Commands run
5. Remaining risks
6. Motion/material QA status
