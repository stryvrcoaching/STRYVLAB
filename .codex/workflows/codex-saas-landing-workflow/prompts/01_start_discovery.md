# Prompt 01 — Start discovery

Use the `landing-workflow-orchestrator` skill.

First audit this repository and identify:
- existing landing page route;
- framework;
- design system;
- components;
- CSS/theme/tokens;
- assets;
- current CTAs;
- existing copy;
- current responsive behavior;
- whether `AGENTS.md` exists.

Then ask me only the missing questions required to create a `LANDING_BRIEF.md`.

Do not implement anything yet.
Do not make visual changes yet.
Do not invent product facts or proof.
Do not invoke Liquid Glass or implementation-level Framer Motion work during discovery. Record only whether the user wants either treatment considered later.

Output:
1. Repo audit summary
2. What is already clear
3. Missing questions, max 8
4. Recommended next step
