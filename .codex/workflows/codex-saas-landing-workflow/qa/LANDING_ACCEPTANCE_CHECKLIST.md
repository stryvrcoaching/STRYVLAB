# Landing Acceptance Checklist

## Strategy

- [ ] Target user is clear.
- [ ] Product category is clear.
- [ ] Main problem is specific.
- [ ] Desired outcome is clear.
- [ ] CTA model is consistent.
- [ ] Pricing/access model is clear.
- [ ] No unsupported claim is present.

## Structure

- [ ] Navbar
- [ ] Hero
- [ ] Proof or honest credibility
- [ ] Problem
- [ ] Pain vs relief or equivalent
- [ ] Solution
- [ ] How it works
- [ ] Product demo/walkthrough/visual
- [ ] Pricing/access/demo booking
- [ ] FAQ
- [ ] Final CTA
- [ ] Footer

## Copy

- [ ] Headline is understandable quickly.
- [ ] Subheading explains mechanism.
- [ ] Benefits are outcomes, not only features.
- [ ] Objections are handled.
- [ ] CTA copy is clear.
- [ ] No generic AI copy.
- [ ] No placeholder text.

## UI

- [ ] Product is visible early.
- [ ] Visual hierarchy is strong.
- [ ] Typography is consistent.
- [ ] Spacing is consistent.
- [ ] Layout feels custom, not template.
- [ ] Cards are used intentionally.
- [ ] Accent color is controlled.
- [ ] Dark/light mode is coherent if applicable.

## Motion

- [ ] Motion has a purpose.
- [ ] Motion does not hide content.
- [ ] Motion does not delay comprehension.
- [ ] Reduced-motion fallback exists.
- [ ] Mobile motion is controlled.
- [ ] No scroll hijacking.
- [ ] Framer Motion is the default React motion layer unless an advanced GSAP sequence is justified.
- [ ] No element or property is controlled by both Framer Motion and GSAP.
- [ ] Motion uses transform/opacity and does not introduce layout shift.

## Material treatment

- [ ] UI/UX Pro Max design-system guidance was reviewed without overriding the approved brand direction.
- [ ] Liquid Glass is absent unless the brief explicitly selected it.
- [ ] If Liquid Glass is active, it is limited to one or two small, non-essential surfaces.
- [ ] Liquid Glass does not cover forms, dense content, screenshots, or all cards.
- [ ] Chromium refraction and Safari/Firefox frosted fallback were checked.
- [ ] Material treatment remains legible, static, and usable with reduced motion.

## Responsive

- [ ] Desktop checked.
- [ ] Tablet checked.
- [ ] Mobile checked.
- [ ] Nav works on mobile.
- [ ] No horizontal scroll.
- [ ] Product visuals remain readable.
- [ ] Pricing/cards stack correctly.
- [ ] CTA remains accessible.

## Accessibility

- [ ] Semantic HTML.
- [ ] Alt text.
- [ ] Focus states.
- [ ] Sufficient contrast.
- [ ] Keyboard accessible interactions.
- [ ] No content only conveyed by color.
- [ ] No autoplay audio.

## Performance

- [ ] Images optimized.
- [ ] Below-the-fold media lazy-loaded.
- [ ] Video compressed or avoided.
- [ ] Poster image used for video if applicable.
- [ ] No unnecessary dependencies.
- [ ] Heavy animation risks reported.

## Engineering

- [ ] Existing stack respected.
- [ ] Components are maintainable.
- [ ] No giant unstructured page component.
- [ ] Content constants separated if useful.
- [ ] Lint run if available.
- [ ] Typecheck run if available.
- [ ] Build run if available.
- [ ] Failures reported honestly.

## Proof integrity

- [ ] No fake testimonials.
- [ ] No fake logos.
- [ ] No fake numbers.
- [ ] No fake compliance.
- [ ] No fake pricing.
- [ ] No fake urgency.
